# AI 多功能互动对话网页设计方案

## 1. 项目概述
本项目旨在开发一个基于 Vue 3 的多功能 AI 对话网页。与传统枯燥的文本对话不同，本项目将引入**“卡通贴纸形态的 AI 助手”**，通过动态气泡框进行回复，并能够根据对话内容自动解析情绪指令，实时切换卡通形象的表情或动作，从而极大提升用户的交互体验和趣味性。

## 2. 核心交互设计

### 2.1 卡通贴纸与气泡框交互
*   **卡通形象展示**：AI 助手以卡通贴纸的形式悬浮或固定在页面一侧（如页面右下角或对话框旁边）。
*   **动态气泡框**：
    *   当 AI 开始回复时，从卡通贴纸的嘴部/头部位置弹出带尾巴的对话气泡框。
    *   气泡框支持打字机效果（Typewriter effect），文本逐字显现。
    *   气泡出现和消失时带有 Q 弹的 CSS 过渡动画（如 `transform: scale` 和 `opacity` 变化）。

### 2.2 情绪指令与图片切换逻辑
为了让 AI 拥有“喜怒哀乐”，我们将采用**指令解析机制**：
*   **系统提示词（System Prompt）约定**：在调用 AI 接口时，要求 AI 在回复文本的开头或特定结构中附带情绪标签。
    *   *示例格式 1 (标签法)*：`[happy] 见到你很高兴！`
    *   *示例格式 2 (JSON法)*：`{"emotion": "happy", "text": "见到你很高兴！"}`
*   **前端监测逻辑**：
    *   接收到 AI 回复后，前端使用正则表达式或 JSON 解析提取情绪指令。
    *   提取后，将纯净的文本传递给气泡框展示。
    *   将情绪指令传递给“状态机”（State Machine），触发卡通贴纸组件更新对应的图片源。
*   **预设情绪状态字典（State Dictionary）**：
    *   `idle` (默认/待机)
    *   `thinking` (思考中/请求中)
    *   `happy` (开心/大笑)
    *   `sad` (委屈/难过)
    *   `angry` (生气)
    *   `confused` (疑惑)

## 3. 技术选型与架构

*   **前端框架**：Vue 3 (Composition API) + Vite (基于现有的 `vue01` 项目)。
*   **样式方案**：原生 CSS / CSS Variables / Flexbox 布局。利用 CSS `transition` 和 `animation` 实现呼吸感和弹跳动画。
*   **组件划分**：
    1.  `App.vue`: 主视图入口，管理全局状态。
    2.  `ChatContainer.vue`: 聊天主界面容器，包含对话历史和输入区域。
    3.  `ChatInput.vue`: 用户输入框组件。
    4.  `AiAvatar.vue`: AI 卡通贴纸组件（负责接收 emotion 状态并展示对应图片）。
    5.  `SpeechBubble.vue`: 气泡对话框组件（负责展示纯文本、打字机动画，并带有指向 Avatar 的 CSS 三角形尾巴）。

## 4. 实施与开发步骤

### 阶段一：静态 UI 与基础组件搭建
1.  准备一套卡通形象的切图或 SVG 素材，命名如 `avatar-idle.png`, `avatar-happy.png` 等。
2.  开发 `AiAvatar.vue`，实现根据传入的 `emotion` prop 切换图片。
3.  开发 `SpeechBubble.vue`，使用 CSS 伪元素（`::before` 或 `::after`）实现指向贴纸的气泡尾巴。
4.  完成基础的聊天输入与展示布局。

### 阶段二：逻辑实现与动画打磨
1.  实现消息发送逻辑与模拟的 AI 延迟回复。
2.  编写“情绪提取器”工具函数（Regex 解析）。
3.  实现状态联动：发送消息时 AI 切换为 `thinking`，解析到指令后切换为对应情绪，并在气泡框中逐字显示回复。
4.  添加 CSS 动画（如贴纸的上下浮动“呼吸”效果，气泡弹出的回弹效果）。

### 阶段三：真实 AI 接口接入
1.  对接真实的 LLM API（如 OpenAI, 智谱, 豆包 等）。
2.  编写 System Prompt，强制规范 AI 的输出格式以配合前端的情绪系统。

## 5. 预期效果示例

**User**: "今天天气真差，我又迟到了。"
**AI API 原始返回**: `[sad] 哎呀，听起来真是糟糕的一天，抱抱你。`
**前端处理**: 
1. 提取情绪 `sad`，`AiAvatar` 组件立即切换为哭泣/委屈的卡通图片。
2. 气泡框从卡通人物嘴边弹出，开始逐字打印："哎呀，听起来真是糟糕的一天，抱抱你。"
